# 和牌率、放铳率、期望点与局收支工具方案

本文设计一组可复用的局面评估工具，用于在切牌、立直、副露、防守和终局策略中提供更统一的量化信号：

- 和牌率：当前行动后，本局最终由自家和牌的概率。
- 放铳率：当前行动后，本局最终由自家打出的牌被他家荣和的概率。
- 期望点：当前行动后，本局点棒增减的期望值，不含或可选含排名价值。
- 局收支：综合自摸、荣和、放铳、被自摸、横移、流局听牌罚符、供托和本场后的期望点棒变化。

## 目标

第一阶段目标不是做完整的四人博弈搜索，而是提供一个稳定、可解释、可测试的估算层，能被现有 `src/strategy/` 决策逻辑调用。

核心要求：

- 对每个候选动作返回同口径指标，避免不同策略模块各自写一套分散权重。
- 能复用现有 `src/hand/paili.ts`、`src/scoring/`、`src/strategy/evaluators/` 的牌形、进张、价值和安全度能力。
- 在信息不完整时给出近似估算，并显式返回置信度、假设和主要误差来源。
- 支持快速模式用于每次切牌实时推荐，支持深度模式用于离线分析和测试。

## 输入模型

`EstimateRoundInput` 应尽量贴近现有 `GameState` 和 `NanikiruContext`，但补齐概率估算需要的字段：

```ts
interface EstimateRoundInput {
  state: GameState;
  actor: PlayerId;
  action: CandidateAction;
  candidates?: CandidateAction[];
  mode?: "fast" | "balanced" | "deep";
  objective?: "point" | "placement";
  assumptions?: EstimateAssumptions;
}
```

`EstimateAssumptions`：

- `remainingDraws`：剩余自摸巡数。未传时从牌山和副露估算。
- `knownTiles`：自家手牌、河牌、副露、宝牌指示牌、已公开杠牌。
- `opponentProfiles`：他家是否立直、是否副露、是否染手、是否高打点、是否明显弃和。
- `rules`：赤宝、返点、头跳、流局罚符、供托和本场规则。
- `sampling`：深度模式下的采样次数、随机种子和超时。

## 输出模型

```ts
interface RoundEstimate {
  action: CandidateAction;
  winRate: ProbabilityEstimate;
  dealInRate: ProbabilityEstimate;
  expectedAgariPoints: PointEstimate;
  expectedRoundIncome: PointEstimate;
  breakdown: RoundIncomeBreakdown;
  confidence: "low" | "medium" | "high";
  assumptions: string[];
  warnings: string[];
}
```

`breakdown` 建议拆到足够细，便于解释系统展示：

```ts
interface RoundIncomeBreakdown {
  selfRon: number;
  selfTsumo: number;
  dealIn: number;
  opponentTsumoLoss: number;
  otherRonLoss: number;
  exhaustiveDraw: number;
  riichiStick: number;
  honba: number;
  placementBonus?: number;
}
```

其中 `expectedAgariPoints` 只统计自家和牌时的平均得点，`expectedRoundIncome` 统计完整局收支。

## 计算总流程

对每个候选动作执行：

1. 应用候选动作得到行动后局面，例如切出某张牌、立直切牌、副露后打牌。
2. 计算自家向听、有效牌、可见枚数、二层进张、预计打点。
3. 估算他家速度、危险度和打点分布。
4. 分别估算自家和牌率、放铳率、他家自摸率、横移率、流局率。
5. 用得点分布和概率分布合成期望点与局收支。
6. 返回指标、拆分项和解释用原因。

## 和牌率估算

### 快速模式

快速模式使用现有牌理结果和剩余巡数做闭式近似，适合实时排序。

基础公式：

```text
winRate = reachTenpaiRate * agariAfterTenpaiRate * interruptionFactor
```

按向听拆分：

- `0 向听`：直接用有效牌枚数、剩余自摸巡数和副露/立直状态估算和牌概率。
- `1 向听`：估算先听牌概率，再估算听牌后的和牌概率。
- `2 向听`：复用二层进张估算，先得到进入一向听/听牌的概率，再折算。
- `3 向听以上`：只给低精度估算，主要用于区分彻底弃和和仍有进攻价值的手牌。

听牌后自摸/荣和概率：

```text
drawHitRate = 1 - product(1 - ukeireAfterVisibility / unknownWallSize over ownDraws)
ronHitRate = opponentDiscardOpportunity * expectedRonAcceptance
```

听牌后的核心输入不应只是理论枚数，而应先计算“有效和牌枚数”：

```text
effectiveWaits =
  sum(remaining(tile) * tileWinWeight(tile) * shapeWeight(waitShapeForTile))

agariAfterTenpaiRate =
  1 - (1 - effectiveWaits / unknownWallSize) ^ effectiveTurnCount
```

其中：

- `remaining(tile)` 使用真实剩余枚数，由 4 枚扣除自家手牌、河牌、副露、宝牌指示牌和其他可见牌。
- `tileWinWeight` 描述该待牌是否容易被打出。
- `shapeWeight` 描述该待牌来自哪类听牌形。
- `effectiveTurnCount` 把自摸机会和荣和机会折成等效巡数，受剩余巡目、立直、默听、副露和对手防守倾向影响。

### 最终听牌有效枚数

和率主要由四类因素共同决定：

1. 听牌种类。
2. 听牌形。
3. 真实剩余枚数。
4. 剩余巡目。

听牌种类权重建议先用启发式表，并保持可配置：

| 待牌类型 | 基础权重 | 说明 |
| --- | ---: | --- |
| 现 2 字牌 | 1.25 - 1.45 | 理论剩余枚数低，但荣和接受度高。 |
| 现 1 字牌 | 1.10 - 1.25 | 比生张字牌更容易被处理。 |
| 普通生张字牌 | 1.05 - 1.15 | 与幺九数牌接近，终盘生张需下调。 |
| 幺九数牌 | 1.05 - 1.15 | 外侧牌，通常较容易出。 |
| 2/8 | 0.95 - 1.05 | 接近中性。 |
| 3/7 | 0.85 - 0.95 | 中张偏难。 |
| 4/6 | 0.78 - 0.90 | 更容易被扣。 |
| 5 | 0.70 - 0.82 | 中心牌，荣和权重较低。 |
| 宝牌 | 0.35 - 0.55 | 他家通常不易打出。 |
| 宝牌周边 | 0.60 - 0.80 | 兼具危险度和价值，荣和权重下调。 |

自家牌河筋可以提高荣和权重，但只作为待牌种类修正，不改变真实剩余枚数：

| 自河筋待牌 | 额外修正 |
| --- | ---: |
| 1/9 | +0.20 左右 |
| 2/8 | +0.14 左右 |
| 3/7 | +0.08 左右 |
| 4/5/6 | +0.03 或不加 |

壁和薄壁同样只提高荣和接受度，不弥补枚数不足：

- 例如 `8p` 已见 3 枚时听 `9p`，`9p` 的出和权重上升。
- 例如 `3s` 已见 4 枚时听 `2s`，`2s` 的出和权重明显上升。
- 如果待牌本身只剩 1 枚，壁权重不能把它评成高枚数好听。

听牌形权重应克制，不能压过真实剩余枚数和待牌种类：

| 听牌形 | 建议权重 | 说明 |
| --- | ---: | --- |
| 多面/复合 | 1.15 - 1.35 | 主要来自覆盖待牌多和真实枚数更容易高。 |
| 两面 | 1.00 | 标准基准。 |
| 双碰 | 0.90 左右 | 同为 4 枚时不应显著弱于两面，关键看待牌种类。 |
| 坎张 | 0.80 左右 | 主要弱在理论上限 4 枚。 |
| 边张 | 0.80 左右 | 与坎张接近，外侧属性另由待牌种类加权。 |
| 单骑 | 0.65 左右 | 理论最多 3 枚，强弱高度依赖待牌种类。 |

需要特别注意双碰的剩余枚数上限：

```text
双碰每种待牌自己已持有 2 枚，因此每种最多只剩 2 枚。
两个待牌都未见时合计最多 4 枚。
一个现 1、一个未见时合计最多 3 枚。
两个都现 1 时合计最多 2 枚。
现 2 字牌若已经外面见 2 枚，该字牌本身已经没有剩余枚数。
```

因此“现 2 字牌容易出”只能提高该牌的荣和权重，不能误算出额外剩余枚数。

一向听和率应枚举进张后的最终听牌形，而不是只看当前进张数：

```text
iishantenWinRate =
  sum(P(draw = t) * bestAgariRateAfterTenpai(afterDraw(t)))
```

这能区分“进张多但多转坎边/单骑”和“进张略少但大量转两面/多面”的候选。形状收益主要应体现在听牌前的未来听牌质量期望；已经最终听牌且真实枚数相同的情况下，听牌形本身只给小幅修正。

其他需要考虑的修正：

- 立直后荣和率高于默听的暴露度收益，但会增加对手防守、被追立和后续放铳风险。
- 副露手荣和率受役确定性影响，没有役或只能自摸役时应降低。
- 役牌生张、场风、自风和三元牌在自家副露或染手明显时应下调。
- 终盘仍然生张的字牌不应一直按容易出处理，尤其是役牌和对危险副露相关的字牌。
- 宝牌和高打点会提高他家不押牌时的防守倾向，可能降低荣和率但也降低放铳率。

### 深度模式

深度模式做有限宽度随机模拟：

- 根据未知牌池生成若干牌山样本。
- 对自家使用当前策略模块选择后续切牌。
- 对他家使用 `OpponentModel` 生成摸切、副露、立直、弃和行为。
- 模拟到自家和牌、他家和牌或流局。

深度模式输出用于校准快速模式，不要求每次线上决策都启用。

## 放铳率估算

放铳率分为当前切牌即时放铳和后续巡目累计放铳。

```text
dealInRate = immediateDealInRate + survivalRateAfterDiscard * futureDealInRate
```

### 当前切牌危险度

复用或扩展 `evaluate-defense.ts` 的安全度信号：

- 现物：对对应立直家近似 0。
- 筋牌：按外侧/中筋/跨筋区分。
- 壁：四枚见、三枚见、单壁降低危险。
- 字牌：生张、初牌、客风、役牌分别处理。
- 早外、早切、手出摸切信息：有数据时参与。
- 副露者：结合副露类型、染手可能、役牌碰、宝牌可见性和切牌序列。

对每个对手计算 `dangerToOpponent`，再合成当前切牌即时放铳率：

```text
immediateDealInRate = 1 - product(1 - dangerToOpponent[i])
```

### 通过筋组压缩

生张无筋的放铳率应随场上已经通过的筋组增加而上升。原因是安全信息越多，剩余未被证明安全的生张中张越容易承载对手待牌。

基础做法：

```text
passedSujiGroups = 已通过筋组数量
totalSujiGroups = 18
globalCompression = passedSujiGroups / totalSujiGroups

sameSuitCompression = passedSujiGroupsInSuit(tile.suit) / 6

sujiCompressionFactor =
  1
  + globalCompression * 0.35
  + sameSuitCompression * 0.45
  + lateTurnBonus

adjustedDanger =
  baseDanger * sujiCompressionFactor
```

每色 6 组筋按 `14`、`25`、`36`、`47`、`58`、`69` 统计。对某个对手来说，若其河里出现了某张牌，则对应两面筋组可视为“已通过”或部分通过；同色通过筋对同色生张的修正更强。

该修正主要作用于：

- 生张无筋中张。
- `4/5/6`。
- 宝牌和宝牌周边。
- 跨筋危险牌。

该修正不应作用或只应极弱作用于：

- 现物。
- 明确筋牌。
- 三见字牌、四见字牌。
- 明确壁支撑的外侧牌。

示例：

```text
早中巡，已通过 3/18 组筋：
  无筋 5m 放铳率约 12% * 1.13

终盘，已通过 12/18 组筋：
  无筋 5m 放铳率约 12% * 1.53
```

实现上该因子应作为乘法修正，避免和“无筋 5”“宝牌周边”等基础危险度重复加算。多家威胁时，仍然先分别计算 `dangerToOpponent`，再用独立事件合成：

```text
immediateDealInRate = 1 - product(1 - adjustedDangerToOpponent[i])
```

### 后续累计放铳

后续放铳率依赖是否继续进攻：

- 进攻路线：按预计还需要切出的危险牌数量累计。
- 防守路线：按可用安牌数量、共通安牌数量和剩余巡数估算。
- 立直路线：立直后无法换牌，后续放铳率应高于默听可回避路线。

建议返回两个子指标：

- `dealInRateIfPush`：继续推进时的累计放铳率。
- `dealInRateIfFold`：从下一巡开始转防守的累计放铳率。

策略层再根据当前目标选择使用哪个。

## 期望点估算

期望点需要先得到自家和牌时的得点分布。

### 手牌价值

优先复用 `src/scoring/`：

- 已听牌或准听牌：枚举可能和牌牌，调用 `calculateAgariScore`。
- 一向听/二向听：枚举主要进张路径，估算最终听牌形和役种。
- 更远向听：使用 `routes.ts` 的 `RoutePortfolio`、宝牌/路线潜力、门清/副露状态进行粗估；value evaluator 只负责把 portfolio 与动态打点估算折算成分数。

`HandValueDistribution`：

```ts
interface HandValueDistribution {
  ron: Array<{ points: number; probability: number }>;
  tsumo: Array<{ gain: number; probability: number }>;
  averageRon: number;
  averageTsumoGain: number;
  manganRate: number;
  limitHandRate: number;
}
```

注意区分：

- 亲家和子家的收支不同。
- 自摸时对每家收入不同，但局收支对自家可直接用总收入。
- 供托和本场只在自家和牌时加到自家收入；放铳和被自摸时按规则扣点。
- 立直宣言棒应作为当前动作的确定成本，同时在自家和牌时可能回收。

## 局收支计算

局收支使用概率加权：

```text
roundIncome =
  P(selfRon) * E(selfRonGain)
  + P(selfTsumo) * E(selfTsumoGain)
  - P(dealIn) * E(dealInLoss)
  - P(opponentTsumo) * E(opponentTsumoLoss)
  + P(otherRon) * E(otherRonDelta)
  + P(draw) * E(drawDelta)
  + deterministicCosts
```

其中：

- `otherRonDelta` 通常为 0，但需要考虑亲被连庄、供托流向和南四排名目标时的间接价值。
- `drawDelta` 根据听牌人数分配罚符；自家听牌时正收益，未听牌时负收益。
- `deterministicCosts` 包含立直棒支出、副露后的防守价值损失等能直接确定的成本。

第一阶段可先按点棒收支实现。第二阶段再加入排名价值：

```text
placementEV = sum(P(finalPlacement = k) * umaOkaValue[k])
```

排名价值主要用于南三、南四、避四、争一位和二位保持。

## 对手模型

`OpponentModel` 不需要一开始很复杂，但必须统一输出：

```ts
interface OpponentEstimate {
  tenpaiRate: number;
  winRate: number;
  tsumoRate: number;
  ronRateAgainstActor: number;
  expectedRonValue: number;
  expectedTsumoLossToActor: number;
  pushAgainstRiichiRate: number;
}
```

输入信号：

- 巡目。
- 立直状态。
- 副露数量和副露类型。
- 打点线索：宝牌副露、役牌碰、染手、断幺/役牌/混一色可能。
- 河牌速度：早期中张切出、孤立牌处理、是否明显手牌很整。
- 点棒和排名动机：亲家、南场、ラス目进攻倾向。

第一阶段可以用规则表和权重。后续如果有牌谱数据，再把这些字段作为特征训练校准模型。

## 与现有策略的关系

现有策略中已有若干局部信号：

- `evaluate-route.ts`：消费 before/after `RoutePortfolio`，评价路线清晰度、强化和破坏。
- `routes.ts`：静态役种路线、宝牌路线、复合路线和冲突关系。
- `evaluate-value.ts`：消费 `RoutePortfolio`、听牌实算和二层打点估算，输出打点潜力。
- `evaluate-defense.ts`：安全度和防守风险。
- `placement.ts`：排名和点差策略。
- `choose-action.ts`：最终动作选择。

新工具不应替代这些模块，而是把它们合成为统一输出：

```text
candidate action
  -> route/value/defense/placement evaluators
  -> round estimate
  -> policy scoring
  -> explanation
```

策略层可以逐步从“分散加权分数”迁移到“局收支 + 排名修正 + 规则保护”的结构。

## 落地阶段

### 第一阶段：确定性估算

实现 `win-rate.ts`、`deal-in-rate.ts`、`hand-value.ts` 和 `round-income.ts` 的快速模式。

验收标准：

- 听牌有效牌更多时，和牌率单调上升。
- 真实剩余枚数相同的最终听牌，听牌形只产生小幅差异；待牌种类、壁/薄壁、自河筋和宝牌修正能改变排序。
- 双碰剩余枚数按每种待牌最多 2 枚计算，现 1/现 2 字牌只提高荣和权重，不增加剩余枚数。
- 现物放铳率低于筋牌，筋牌低于无筋中张。
- 同一张生张无筋中张，在通过筋组更多、同色通过筋更多、巡目更晚时，放铳率上升。
- 满贯手的期望点高于 1000 点愚形手，但在高放铳率下局收支可能更低。
- 立直动作正确扣除 1000 点，并在自家和牌时计入供托回收。

### 第二阶段：和现有推荐系统接入

在 `evaluate-nanikiru.ts` 或 `choose-action.ts` 中加入可选估算结果：

- 默认 `fast`，不影响现有输出速度。
- `--include-estimate` 时在解释中展示和牌率、放铳率、期望点和局收支。
- 原有策略分数仍保留，先作为对照。

#### EV 参与决策的具体方式

EV 不直接替代原有策略评分。原有评分继续负责牌理、路线、改良、防守资源和场况规则，EV 作为同口径的局收支信号参与二次仲裁，并以低权重进入候选分数拆分。

第一版采用三层结构：

1. `scoreBreakdown.ev`：每个候选都挂载 EV 分数，便于解释和调试。点棒局收支按低权重折算：

```text
evScore = expectedRoundIncome / 100
```

该分数不直接压过向听差。向听差仍由原策略保护；EV 只在同向听或允许退向的候选之间影响排序。

2. EV 二次仲裁：先按原策略完成排序，再在前若干候选中寻找可仲裁对象。只有满足以下条件才允许 EV 改变推荐：

- 候选处于同一向听，或退向候选已经被原有防守/早巡改良规则允许参与推荐。
- 原策略分差处于同一决策带内：

```text
abs(scoreA - scoreB) <= max(60, abs(scoreA) * 0.12)
```

- 至少一个候选的 `RoundEstimate.confidence` 不是 `low`，且两者都有 EV 输出。
- EV 局收支差距足够显著：

```text
expectedRoundIncomeDelta >= 250
```

若满足条件，按 `expectedRoundIncome` 重排同一决策带内候选。

3. EV 保护规则：当候选存在明显风险时，允许 EV 做强约束，但必须保守：

- 若某候选放铳率比另一候选高至少 `0.05`，且局收支低至少 `600` 点，不推荐该高风险候选。
- 防守/平衡模式下，若某候选局收支领先至少 `400` 点且放铳率不更高，可以覆盖小幅牌效劣势。
- 进攻/推进模式下，EV 只在分差接近时仲裁；不允许为了小幅局收支牺牲明确向听优势。

`decide` 使用完整 `GameState`，默认启用 EV 决策参与。`nanikiru` 常只有手牌和简化上下文，因此使用从手牌、宝牌、场风、自风、副露和已知可见牌合成的简化 `GameState`，同样启用 EV 仲裁，但在缺少对手河牌与威胁信息时置信度较低，解释中应保留“估算”语气。

CLI 行为：

- `npm run decide -- --state examples/state-east1.json`：推荐动作默认已经过 EV 二次仲裁。
- `npm run decide -- --state examples/state-east1.json --include-estimate`：额外输出推荐候选的 EV 明细。
- `npm run nanikiru -- "..."`：在简化局面下使用 EV 二次仲裁。
- 若需要仅看原策略，可增加关闭开关，例如 `--no-ev-decision`。

验收标准：

- 推荐解释能说明“和牌率更高但放铳风险过大，所以选择防守”。
- 南四避四场景能展示局收支和排名目标的差异。
- 原有测试不因接入估算层产生大规模快照波动。

### 第三阶段：采样模拟与校准

实现 `deep` 模式：

- 从未知牌池采样。
- 使用当前策略模拟自家后续动作。
- 使用简化对手模型模拟他家行为。
- 输出均值、方差和置信区间。

验收标准：

- 同一随机种子输出稳定。
- 采样次数增加时结果波动下降。
- 快速模式与深度模式在常见牌姿下方向一致。

### 第四阶段：排名 EV

将点棒收支转换为排名价值：

- 支持南三、南四、亲番剩余、返点、乌马、头名奖励。
- 计算目标点差，例如逆转、保二、避四。
- 对局收支增加 `placementBonus`。

验收标准：

- 南四领先时，低局收支但高避险动作可能优于进攻动作。
- 南四落后时，高打点低和率路线可能优于小和路线。
- 流局听牌罚符对避四和连庄判断有明确影响。

## 测试策略

单元测试：

- 牌山可见枚数变化对和牌率的影响。
- 最终听牌有效枚数：普通生张字牌与幺九数牌接近，宝牌和宝牌周边下调，现 1/现 2 字牌提高荣和权重但不增加剩余枚数。
- 听牌形权重：同为 4 枚时两面与普通双碰差异有限；多面优于两面；单骑因理论上限和形状权重较低。
- 一向听二层和率：进张后多转两面/多面的候选，应能优于进张略多但多转坎边/单骑的候选。
- 不同安全度牌的放铳率排序。
- 通过筋组压缩：生张无筋中张的放铳率随全局和同色通过筋组增加而上升，现物和明确筋牌不受该乘法修正影响。
- 亲/子、荣和/自摸、本场/供托的收支计算。
- 立直棒成本和回收。
- 流局听牌人数罚符分配。

性质测试：

- 所有概率必须在 `[0, 1]`。
- 同一事件分解的概率和不能明显超过 1。
- 所有候选动作使用同一组规则和假设。
- 对称局面下，不同等价牌的估算应接近。

回归测试：

- 固定若干实战局面 JSON。
- 对每个局面记录候选动作排序、关键指标和解释。
- 调整权重时必须能看到排序变化原因。

## 风险与边界

- 没有牌谱数据时，放铳率和他家和牌率只能是启发式估算，必须在输出中标注 `confidence`。
- 快速模式不应伪装成精确 EV。解释文案应使用“估算”“倾向”“约”等表达。
- 他家手牌不可见，副露和立直场景的误差会显著小于门清未立直场景。
- 高级役、特殊牌形、海底/岭上/抢杠等偶发事件第一阶段可以忽略或折成微小修正。
- 排名 EV 与规则强相关，必须明确乌马、返点、头跳、南入等规则输入。

## 推荐优先级

建议先实现点棒口径的快速模式：

1. `HandValueDistribution`：枚举听牌/一向听主要和牌牌并复用 `src/scoring/`。
2. `estimateWinRateFast`：基于向听、有效牌、剩余巡数和听牌形。
3. `estimateDealInRateFast`：基于当前切牌危险度和后续推进/防守路线。
4. `estimateRoundIncome`：把概率和得点合成局收支。
5. CLI 和解释接入：让用户能看到每个候选动作的四个核心指标。

这条路径能最快把“和牌率、放铳率、期望点、局收支”从抽象策略词变成可调试、可比较、可测试的工具输出。
