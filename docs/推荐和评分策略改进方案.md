# 推荐和评分策略改进方案

本文记录已经落地的麻将决策逻辑与解释系统改进。

## 1. 决策模式调整

目标：一向听、高打点手牌在面对威胁时更偏向进攻，而不是固定进入 `balance`。

当前规则：

```ts
function chooseStrategyMode(state: GameState, shanten: number): StrategyMode {
  const hasRiichiThreat = state.opponents.some((opponent) => opponent.riichi);
  if (!hasRiichiThreat) return "attack";
  if (shanten >= 2) return "defense";
  if (shanten === 0) return "push";
  if (shanten === 1 && isHighValueHand(state)) return "push";
  return "balance";
}
```

高打点手牌判定：

```ts
function isHighValueHand(state: GameState): boolean {
  const doraTiles = state.doraIndicators.map(nextDoraTile);
  const selfTiles = [...state.self.hand, ...(state.lastDraw ? [state.lastDraw] : [])];
  const doraCount = selfTiles.filter((tile) => doraTiles.includes(tile)).length;

  return doraCount >= 2 || state.averageWaitPoints >= 7700;
}
```

目前代码中 `GameState` 标准类型尚未声明 `averageWaitPoints`，实现会兼容读取该可选字段；若没有该字段，则只按宝牌数量判定。

## 2. 防守权重动态调整

目标：高打点手牌时降低过度防守对牌效和打点的压制。

当前模式权重：

```ts
if (mode === "balance") {
  defenseWeight = highValueHand ? 2 : 5;
}

if (mode === "push") {
  defenseWeight = 2;
  valueWeight *= 1.15;
}
```

即一向听高打点面对立直会进入 `push`，保留一定防守风险评估，但不会像 `defense` 模式一样强行弃和。

## 3. 防守余力 / 后续防守空间

目标：避免切掉唯一安全牌，导致“攻不了也防不了”。

当前防守综合分：

```text
defenseScore =
  immediateSafetyMinusDanger(discard)
  + defenseReserveScore(remainingTiles)
  - futureDefensePenalty(safeTileCountAfterDiscard)
```

`defenseReserveScore` 对切牌后的剩余手牌计分：

```ts
function defenseReserveTileScore(tile: Tile): number {
  if (isGenbutsu(tile)) return 100;
  if (isSujiTerminal(tile)) return 45; // 1/9 筋
  if (isSuji28(tile)) return 35;       // 2/8 筋
  if (isSuji456(tile)) return 20;      // 4/5/6 筋
  if (isSuji37(tile)) return 10;       // 3/7 筋
  return 0;
}
```

“较可靠防守牌”当前按安全分 `>= 35` 计入，也就是现物、`1/9` 筋和 `2/8` 筋。`3/7` 筋不会被算作较可靠防守资源。

后续防守不足惩罚：

```ts
function futureDefensePenalty(safeTileCountAfterDiscard: number): number {
  if (safeTileCountAfterDiscard === 0) return 220;
  if (safeTileCountAfterDiscard === 1) return 120;
  if (safeTileCountAfterDiscard === 2) return 50;
  return 0;
}
```

## 4. 筋牌安全度分级

目标：弱筋不要被高估，尤其是 `3/7` 筋。

当前筋牌安全度按以下顺序逐渐降低：

```ts
function sujiSafetyScore(tile: Tile, isGenbutsu: boolean): number {
  if (isGenbutsu) return 100;
  if (isSujiTerminal(tile)) return 45; // 1/9
  if (isSuji28(tile)) return 35;       // 2/8
  if (isSuji456(tile)) return 20;      // 4/5/6
  if (isSuji37(tile)) return 10;       // 3/7
  return 0;
}
```

这样 `3m/7m` 这类弱筋只提供很小的安全补正，不会被误判为强安全牌。

## 5. 推荐理由系统改进

目标：

- “理由”只显示支持推荐动作的非负面理由。
- 高打点理由优先级提升。
- 负面理由单独展示为“注意”。

当前解释层流程：

```ts
const reasons = normalizeReasonPriorities(best.reasons)
  .filter((reason) => reason.polarity !== "negative")
  .sort((a, b) => b.priority - a.priority)
  .slice(0, 4);

const warnings = best.reasons.filter((reason) => reason.polarity === "negative");
```

高打点优先级：

```ts
if (points >= 12000) priority = 92;
else if (points >= 7700) priority = 86;
else priority = Math.max(priority, 64);
```

## 6. 比较型解释

目标：解释“为什么选这个动作而不是另一个”。

当前已支持两类比较：

- 进张比较：推荐候选进张数高于第二候选时说明进张优势。
- 打点路线比较：推荐候选打点路线优于第二候选时说明路线差异。
- 防守比较：推荐候选进张较少但防守分更高时，说明牺牲部分进张换取风险下降和后续防守资源。
- 牌效接近时的危险中张/外侧牌比较：解释为什么先处理中张，或为什么选择当前和转听牌时压力更低的外侧牌。

这些比较现在集中在 `src/strategy/arbitration.ts`，不再散落在 value 或 route evaluator 中。

防守比较示例：

```text
相比切 7s，切 4z 牺牲部分进张但显著降低对威胁者风险并保留后续防守资源。
```

## 7. 门清听牌后的立直判断

目标：何切推荐如果切后门清听牌，先根据场况、巡目、当前听牌质量和改良价值判断路线，再按路线选切牌。这样避免“推荐默听改良切牌，却又提示立直”的策略冲突。

每个听牌候选仍会返回立直倾向：

```ts
interface RiichiJudgment {
  canRiichi: boolean;
  shouldRiichi: boolean;
  score: number;       // -100 到 100
  level: RiichiLevel;
  levelText: string;   // 强烈建议立直 / 建议立直 / 可立可不立 / 不太建议立直 / 强烈反对立直
  confidence: number;  // 0 到 1
  reasons: string[];
  details: {
    damaAveragePoints: number;
    riichiAveragePoints: number;
    pointGain: number;
    totalWaits: number;
    goodShapeRatio: number;
    allWaitsHaveYaku: boolean;
    anyWaitHasYaku: boolean;
    improvementTiles: number;
    improvementDraws: TileId[];
    shapeImprovementTiles: number;
    valueImprovementTiles: number;
    valueImprovementDraws: TileId[];
    bestImprovedWaits: number;
    bestImprovedDamaPoints: number;
    improvementTurnMultiplier: number;
    uraEstimate: {
      indicatorCount: number;
      expectedUraHan: number;
      hitRate: number;
      multiHitRate: number;
      expectedScoreGain: number;
      upgrade: "none" | "mangan" | "haneman" | "baiman_or_better";
      upgradeProbability: number;
      scoreBonus: number;
      reasons: string[];
    };
    remainingTurns: number;
  };
}
```

整手分析还会返回路线决策：

```ts
interface RiichiPlanDecision {
  plan: "none" | "dama_improvement" | "dama" | "riichi_now";
  shouldRiichi: boolean;
  score: number;
  levelText: string;
  damaDiscard?: TileId;
  riichiDiscard?: TileId;
  reasons: string[];
}
```

等级映射：

```text
score >= 70     强烈建议立直
score >= 35     建议立直
score > -20     可立可不立
score > -55     不太建议立直
score <= -55    强烈反对立直
```

当前判断分四层：

1. 前置条件：必须门清、切后听牌、有有效待牌、点数不少于 1000。副露手、非听牌候选或无立直棒点数时直接不给立直建议。
2. 牌面收益：比较默听和立直后的每种待牌实算点数，按剩余枚数加权出平均打点；同时检查默听是否有役、是否所有待牌都有役。
3. 默听改良：对低打点窄听，模拟整副手牌的可摸牌，检查是否能保持听牌并改善待牌枚数、好形质量、默听有役或平均打点。形状改良优先，打点改良作为次级因素。
4. 里宝期望：对每个立直和牌形统计 14 张牌的牌种、对子和刻子，按宝牌指示牌数量估算里宝命中率、多枚中里率、期望点数增益和跨档概率。
5. 局况和风险：根据听牌枚数、好形率、剩余巡目、对手威胁、南场落后、终局领先等因素修正。

路线决策顺序：

```text
1. 先找最佳默听候选 damaDiscard：默认使用综合何切/EV 后的听牌候选，偏向保留改良。
2. 再找最佳立直候选 riichiDiscard：按立直倾向分、待牌枚数、好形率和立直平均打点评价。
3. 如果最佳立直候选达到立直阈值，选择 riichi_now，并把 riichiDiscard 作为最终推荐。
4. 如果没有达到立直阈值，但默听候选仍有足够改良，选择 dama_improvement，并把 damaDiscard 作为最终推荐。
5. 解释中同时给出另一条路线的候选，例如“如果必须立即立直，切 1s 是更好的立直候选”。
```

巡目对改良价值有衰减：

```text
turn <= 8     改良价值 100%
turn <= 11    改良价值 70%
turn <= 14    改良价值 35%
turn >= 15    改良价值 0%
```

因此同一手牌早巡可能选择默听改良，晚巡则可能切换成立即立直路线。

主要加分：

```text
默听无役：+60
部分待牌默听无役：+35
立直前未满贯：+25
立直平均打点提升 2000+：+20
立直平均打点提升 3900+：+30
听牌枚数 >= 8：+25
听牌枚数 5-7：+15
好形率 >= 60%：+20
南场落后较多：+30
四位追分：+30
立直后达到顺位逆转需求：+40
```

主要减分：

```text
默听满贯以上且所有待牌有役：-25
默听跳满以上且所有待牌有役：-40
听牌枚数 <= 2：-20
剩余巡目 <= 3：-20
剩余巡目 <= 1：-100
一个明显威胁者：-30
多个明显威胁者：-60
终局附近领先：-40
终局附近大幅领先：-70
低打点窄听有充足默听改良：最多 -65，按巡目衰减
```

里宝加分是概率补正，不直接覆盖形状评价：

```text
宝牌指示牌 >= 2：+3
期望里宝 >= 0.7 番：+6
期望里宝 >= 0.35 番：+3
多枚中里率 >= 12%：+4
期望点数增益 >= 1000：+8
期望点数增益 >= 500：+4
跨满贯/跳满/倍满档：按概率 +2 到 +8
总里宝补正上限：+25
```

立直候选选择仍然优先待牌枚数和形状。也就是说，暗刻手牌有更高里宝爆发潜力，但如果另一候选明显听得更宽，仍会优先作为“立即立直”候选。

解释渲染：

```text
立直判断：
- 建议立直（50/100）。
- 立直前未到满贯，立直带来的打点提升较重要。
- 听牌枚数多，立直后的和牌率基础较好。
- 手牌里宝命中率约 32%，期望里宝约 0.41 番。

路线判断：
- 当前更适合切 5m 默听等改良。
- 如果必须立即立直，切 1s 是更好的立直候选。
```

## 总结

| 模块 | 改动目的 | 当前修改 |
| --- | --- | --- |
| 决策模式 | 一向听高打点偏进攻 | `shanten=1 && highValueHand -> push` |
| 防守权重 | 避免高打点手牌过度保守 | `balance` 模式动态防守权重：高打点 `2`，普通 `5` |
| 防守余力 | 避免切掉唯一安全牌 | 增加 `defenseReserveScore` 和 `futureDefensePenalty` |
| 筋牌权重 | 弱筋不要被高估 | `1/9=45`、`2/8=35`、`4/5/6=20`、`3/7=10` |
| 推荐理由 | 解释更聚焦 | 正面理由进“理由”，负面理由进“注意”；高打点理由提优先级 |
| 比较型解释 | 增强可理解性 | 支持进张、打点路线、防守资源对比 |
| 立直判断 | 给门清听牌候选返回立直必要性 | 候选新增 `riichiJudgment`，包含倾向分、等级、置信度、打点、改良、里宝和局况原因 |
| 立直路线 | 避免默听切牌和立直切牌冲突 | 分别选择 `damaDiscard` 和 `riichiDiscard`，再按路线决定最终推荐 |
| 策略层重构 | 降低路线和 tie-break 互相缠绕 | `CandidateFeature`、`RoutePortfolio`、`ImprovementEvaluator`、`DecisionArbitrator` 已落地 |
