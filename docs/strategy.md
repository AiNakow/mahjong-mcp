# 策略仲裁设计与工作流程

本文是当前策略层的权威说明，覆盖何切评分、局面模式、合法动作、统一动作仲裁、EV 二次仲裁和解释生成。

## 设计目标

策略层解决两个问题：

1. 在给定手牌和上下文时，给出可解释的何切推荐。
2. 在给定完整 `GameState` 时，生成当前阶段所有合法动作，统一评分后返回推荐动作。

原则：

- 规则合法性和策略评分分离。
- 能和牌时默认优先短路，避免继续运行昂贵分析。
- 何切分数由多个 evaluator 叠加，不由单一进张数决定。
- EV 是估算信号，只在原策略分数接近或规则允许的决策带内参与仲裁。
- 解释只渲染 reasons，不重新计算策略。

## 顶层入口

```ts
chooseAction(state: GameState, options?: ChooseActionOptions): ActionDecision
```

总流程：

```text
GameState
  -> determineDecisionPhase
  -> generateLegalActions
  -> evaluateAgariActions
  -> agari short-circuit if valid
  -> evaluate discard / riichi / call / kan / pass
  -> apply unified EV
  -> arbitrateActions
  -> ActionDecision
```

`ActionDecision` 主要包含：

- `phase`
- `mode`
- `action`
- `candidates`
- `analysis`
- `explanation`

## 决策阶段

当前支持的 `DecisionPhase`：

- `self_draw`：自家摸牌后。
- `opponent_discard`：他家打牌后。
- `chankan`：抢杠响应窗口。
- `rinshan_draw`：岭上摸牌阶段。
- `after_call_discard`：吃碰后必须切牌。
- `unknown`：无法判断或兼容旧输入。

完整 JSON 推荐显式传入 `phase`。轻量 CLI 会根据 `lastDraw`、`lastDiscard` 和手牌形态推导阶段。

## 合法动作生成

`generateLegalActions` 只负责规则合法性，不负责推荐倾向。

### self_draw

生成：

- `tsumo`
- `riichi`
- `ankan`
- `kakan`
- `discard`

立直合法性要求门清、未立直、点数足够、切后听牌且不是不允许立直的阶段。立直后暗杠采用保守规则：只允许自摸牌补成手内暗刻第四张，且暗杠前后待牌集合完全一致。

### opponent_discard

生成：

- `ron`
- `chi`
- `pon`
- `minkan`
- `pass`

吃牌只允许吃上家打出的数牌。立直后不能吃碰明杠。

### chankan

生成：

- `ron`
- `pass`

当前只处理他家加杠产生的抢杠响应。

### rinshan_draw

生成岭上自摸、切牌和可行杠动作。计分时自摸会带 `rinshan` 上下文。

### after_call_discard

只生成副露后合法切牌，并应用食替限制。

## 和牌优先短路

`evaluateAgariActions` 会对合法的 `tsumo` / `ron` 调用计分模块。若存在有效和牌动作，`chooseAction` 默认直接返回和牌候选，不继续运行何切、立直、副露、杠或 EV。

当前振听判断：

- 基础振听：当前所有待牌中任一张出现在自家河里，则不能荣和。
- 临时振听：对可荣和牌选择 `pass` 后进入临时振听，自家摸牌阶段解除。
- 立直见逃振听：立直后对可荣和牌选择 `pass` 后进入立直振听，后续保持。

终局特殊策略可以拒绝无法改善顺位的和牌，但已立直时不会主动见逃。

## 策略模式

`chooseAction` 会先选择基础模式：

- `attack`：无主动威胁。
- `defense`：有立直威胁且自家两向听以上。
- `push`：自家听牌，或一向听且高打点。
- `balance`：有威胁但仍有推进价值。

高打点信号包括：

- 自家宝牌数量达到 2 张以上。
- 或局面中提供的平均待牌点数达到高打点阈值。

模式只调整权重，不改变底层牌理事实。

## 何切评分链路

何切分析入口为 `evaluateNanikiru`，当前调用链：

```text
analyze hand
  -> build CandidateFeature
  -> evaluate RoutePortfolio
  -> evaluate shape / route / value / improvement / defense
  -> apply placement policy
  -> arbitrate final order
  -> optional EV second pass
  -> render explanation
```

候选分项：

- `shanten`：向听优势。
- `ukeire`：总进张。
- `goodShape`：好型相关进张。
- `shape`：形状、块数管理、搭子质量和副露速度价值。
- `route`：路线清晰度、路线强化和路线破坏。
- `value`：打点潜力、听牌实算和一向听二层打点。
- `improvement`：同向听和退向改良。
- `defense`：安全度、危险度和防守余力。
- `ev`：局收支估算折算分。

默认公式是可解释的线性组合，并由 `NanikiruPolicy` 控制权重和阈值。

## CandidateFeature

`features.ts` 为每个候选构造统一事实快照，后续 evaluator 只读 feature，避免重复扫描手牌。

包含：

- 切前/切后手牌。
- 向听、进张、好型进张。
- 对子、刻子、两面、嵌张、边张、复合形等结构。
- 幺九字牌数量、中张数量。
- 宝牌、赤宝牌、宝牌周边。
- 役牌对子。

feature 只描述事实，不直接下推荐结论。

## 路线模型

`routes.ts` 集中管理静态路线识别。新增路线应优先新增 `RouteModel` 并注册到 `ROUTE_MODELS`，而不是在多个 evaluator 中重复写判断。

当前路线：

- `dora`：宝牌、赤宝牌和宝牌周边。
- `yakuhai`：役牌路线。
- `tanyao`：断幺路线。
- `chiitoi`：七对子路线。
- `honitsu`：染手路线。
- `ittsu`：一气通贯路线。
- `sanshoku`：三色同顺路线。
- `chanta_sanshoku`：全带三色复合路线。
- `chanta`：全带路线。
- `toitoi`：对对和路线。

`RoutePortfolio` 会输出候选路线、兼容路线组合线和冲突信息。`evaluate-value.ts` 和 `evaluate-route.ts` 只消费 portfolio，不再重复识别静态役种。

## 形状策略

shape evaluator 覆盖：

- 两面、嵌张、边张。
- 复合形，例如 `2334`、`3445`、`4556`、`5667`、`6778`、`1123`、`7889`。
- 孤立幺九字牌。
- 六搭子/超块时的块数管理。
- 搭子预计听牌质量。
- 副露速度价值。

搭子超载时，策略不会只按当前搭子种类排序，而会估算保留该搭子后更可能形成的最终听牌质量。默认倾向包括：

- 无额外路线约束时，`23` 这类可转外侧两面的两面通常优于 `45`。
- 嵌张/边张比较中，`35 > 79 > 12/89`。
- 役牌对子和明确成役路线会提高可碰资源价值。

副露速度价值考虑：

- 碰可从三家碰，吃只能吃上家。
- 役牌对子速度和成役价值最高。
- 普通幺九对子在已有役或路线明确时，因容易被打出而有较高副露速度价值。
- 无役副露会被压低。

## 打点策略

value evaluator 的打点来源：

- `RoutePortfolio` 中的静态路线。
- 听牌候选的实算最高和牌点。
- 一向听候选的二层转听牌平均打点估算。

听牌实算会枚举待牌，调用 `calculateAgariScore`。一向听二层估算会在预算内枚举有效进张和转听牌切牌，再调用计分模块估算平均点数。

二层估算默认是有限预算：

- 最多若干种有效进张。
- 每种进张后最多若干个转听牌切牌。

它是策略信号，不是完整搜索。

役牌与断幺冲突处理：

- 役牌对子不会无条件压过断幺。
- 若切出役牌对子后手牌明显转向断幺，并提升中张延展和副露空间，会给断幺路线额外加分。
- 若手牌仍强烈需要役牌对子成役，则保留役牌对子。

## 改良策略

`improvement.ts` 负责：

- 同向听改良。
- 早巡低价值听牌退向改良。

同向听改良只在早巡、无主动威胁，并且存在宝牌或明确复合路线信号时启用。

早巡低价值听牌退向改良条件较窄：

- 当前听牌。
- 早巡。
- 没有立直或一发威胁。
- 当前是低价值窄听。
- 退向后一向听有足够进张和好型改良。
- 有宝牌指示牌时，退向后的有效进张应保留实际宝牌进张。

满足条件时，退向候选仍按折算后的向听、进张和好型分参与排序，不会被当成普通一向听。

## 防守策略

防守 evaluator 只在存在威胁对手时生效。威胁对手当前定义：

- 已立直。
- 或副露数达到 2 副露以上。

当前安全/危险特征：

- 现物。
- 字牌见张。
- 幺九牌。
- 壁。
- 分级筋：`1/9 > 2/8 > 4/5/6 > 3/7`。
- 无筋中张危险度：`5 > 4/6 > 3/7 > 2/8`。
- 宝牌和宝牌周边风险。
- 一发巡风险。
- 亲家威胁风险。
- 晚巡风险。
- 切牌后的防守余力。
- 后续防守资源不足惩罚。

防守分结构：

```text
defenseScore =
  immediateSafetyMinusDanger
  + defenseReserveScore
  - futureDefensePenalty
```

当前未覆盖：

- 早外侧。
- 里筋、跨筋、双无筋细分。
- 副露染手危险色。
- 对手手出/摸切序列。
- 精确铳点期望。

## 局况和点棒策略

`placement.ts` 负责高层局况修正。

输入信号：

- 当前排名。
- 与第一名、上一名、第四名的点差。
- 是否南场、南四或终局附近。
- 巡目。
- 自家是否亲家。
- 威胁者是否亲家。
- 供托和本场。
- 当前向听。
- 手牌是否高价值。

输出：

- `pushBias`
- `defenseWeightMul`
- `valueWeightMul`
- `shantenWeightMul`
- `ukeireWeightMul`
- `avoidFourthGoal`
- `reasons`

当前规则：

- 南场领先较多，提高防守权重。
- 终局附近且自家第一，进一步提高防守权重。
- 南场落后较多，提高打点和推进倾向。
- 自家亲家，小幅提高推进收益。
- 威胁者亲家，提高防守权重。
- 供托和本场增加，小幅提高推进收益。

## 南四避四策略

南四三位微差时，策略不会简单等价为防守，而是在以下目标间切换：

- `winOut`：听牌时优先和牌结束。
- `tenpaiKeep`：一向听或中巡时重视保听和速度。
- `fold`：终盘手慢且四位进攻时优先避免放铳。
- `chase`：自家四位时提高推进和打点权重。
- `none`：早巡手慢时避免过早弃和。

关键原则：

- 三位领先四位 4000 点以内时，流局听牌罚符可能改变名次，因此显式提高保听价值。
- 早巡两向听以上不直接转防守。
- 自己四位时目标是追分，不触发防守型避四。

## 立直策略

立直候选由切牌候选生成。切后听牌且满足合法性时，`riichi.ts` 会判断立直、默听或改良。

考虑因素：

- 立直 1 翻和里宝期望。
- 待牌质量。
- 巡目。
- 亲家压力。
- 愚形低打点惩罚。
- 防守锁定代价。
- 南场领先或终局局况。
- 高打点默听价值。

最终立直作为独立 `riichi` 动作参与仲裁，而不是只附加在切牌解释上。

## 副露策略

吃、碰和大明杠评估流程：

```text
合法副露动作
  -> 应用副露得到模拟局面
  -> 副露后运行 open mode 何切评分
  -> 生成 call + discard 完整候选
  -> 与 pass 比较
```

主要信号：

- 是否降向听。
- 是否听牌。
- 是否确定有役。
- 是否获得役牌刻子。
- 是否破坏门清立直价值。
- 副露后打点。
- 副露后切牌危险度。
- 防守资源损失。

倾向：

- 役牌对子被打出且能降向听，通常积极。
- 断幺确定且能降向听，可推荐。
- 混一色路线明确时可推荐。
- 无役、手慢、破坏门清高价值时不推荐。
- 对立直时，非听牌副露明显更保守。

## 杠策略

杠动作包括：

- `ankan`
- `kakan`
- `minkan`

当前原则：

- 大明杠默认保守。
- 对手立直时不轻易开杠。
- 自家听牌或一向听、高打点、落后追分时，杠收益更容易被认可。
- 终局领先时加杠倾向降低。
- 已经可和牌时和牌优先。

杠动作会挂接快速 EV 近似，考虑岭上补一摸、开新宝牌风险和动作类型差异，但不是完整杠后搜索。

## pass 与见逃

`pass` 是他家打牌阶段的合法候选，也是副露和荣和候选的基线。

它表达：

- 不鸣以保留门清和立直价值。
- 不和以追求更高顺位目标。
- 不杠以避免开宝牌风险。
- 对可荣和牌选择不和时写入临时振听或立直见逃振听。

当前仍需要继续增强终局见逃、不鸣和 pass 的解释稳定性。

## EV 二次仲裁

EV 不替代原策略评分。它以三种方式参与：

1. 写入 `scoreBreakdown.ev`。
2. 在原策略分数接近时做二次排序。
3. 在防守/平衡模式下对明显高风险低收益候选做保护。

原则：

- 不覆盖明确向听优势。
- 主要比较同向听或已经被策略允许参与的退向候选。
- 需要 EV 差距足够明显。
- 低置信度估算不会强行改变推荐。

`decide` 默认启用 EV 决策，`nanikiru` 在简化局面下也默认启用，但置信度会受缺少对手河牌和威胁信息影响。可用 `--no-ev-decision` 关闭。

## 动作仲裁

不同动作不能只用何切分直接比较。当前统一候选包含：

- `action`
- `phase`
- `category`
- `score`
- `priority`
- `scoreBreakdown`
- `estimate`
- `reasons`
- `warnings`
- `source`

类别：

- `agari`
- `discard`
- `riichi`
- `call`
- `kan`
- `pass`

和牌有最高基础优先级。立直、切牌、副露、杠和 pass 再通过策略分、局况、EV 和 reasons 仲裁。

## 动作应用和状态推进

`applyDecisionAction` 支持：

- 普通切牌。
- 立直切牌。
- 不鸣/不和。
- 吃碰后的切牌。
- 杠动作登记。
- 和牌终局标记。

状态规则：

- 吃碰后写入食替限制。
- 可荣和牌选择 pass 后写入临时振听。
- 立直后见逃写入立直振听。
- 进入自家摸牌阶段会解除普通临时振听。
- 立直振听不会因摸牌解除。

## reasons 与解释

统一 reason 结构：

```ts
type Reason = {
  type: ReasonType;
  polarity: "positive" | "negative" | "neutral";
  priority: number;
  message: string;
  data?: Record<string, unknown>;
};
```

解释层职责：

1. 选择推荐候选。
2. 挑选高优先级正面或中性 reasons。
3. 将负面 reasons 放到“注意”。
4. 渲染中文文本。

解释层不重新计算牌理、打点、防守或局况。

## 当前边界

暂未实现或仍是基础版：

- 完整对局循环。
- 多人同时荣和结算。
- 包牌。
- 四杠散了。
- 更精确的副露对手读牌。
- 手出/摸切序列。
- 排名 EV 的完整建模。
- 自然语言题面解析。
- 截图识别。

## 后续优先级

建议下一阶段优先：

1. 收口 `pass`、终局见逃和不鸣解释。
2. 增强杠动作边界测试。
3. 继续优化副露、立直计划和二层估值慢测。
4. 抽出共享 JSON Schema 和统一服务 facade，为 HTTP API 和 MCP 做准备。
