# 南四避四与流局听牌策略方案

实现状态：已在 `src/strategy/placement.ts` 和 `src/strategy/choose-action.ts` 落地第一版，并用 `tests/choose-action.test.ts` 覆盖核心场景。

本文记录一个方案：在南四或终局附近，三位与四位微差时，推荐策略不应简单变成防守，而应在“抢和结束”“保听流局”“防放铳”之间切换。

## 背景问题

当前点棒/局况层已经支持：

- 南场领先偏守。
- 南场落后偏攻。
- 终局附近第一提高防守权重。
- 自家亲家提高推进收益。
- 亲家威胁提高防守权重。

但“避四”不能简单等价为“防守”。南四三位只领先四位很少时，很多实战局面里，最安全的避四路线反而是：

- 抢速度和牌结束。
- 或至少保持听牌到流局，避免被不听罚符逆转。

特别是与四位差距 `4000` 点以内时：

- 自己不听、四位听牌，可能被罚符逆转。
- 自己听牌、四位不听，可以拉开点差。
- 双方都听或都不听，通常维持点差。
- 过早弃和可能把主动权交给四位，反而增加落四风险。

因此需要将南四微差避四拆成独立目标，而不是只通过 `defenseWeight` 变大来表达。

## 设计目标

1. 避免“南四三位微差，配牌三向听就开始弃和”。
2. 听牌或一向听时，优先考虑和牌结束或保听。
3. 中后巡手慢且四位有威胁时，才明显转向防放铳。
4. 与四位差距 `4000` 点以内时，显式提高“流局听牌”的价值。
5. 该策略应是局况层，不直接重写底层牌理算法。

## 新增目标类型

建议在 `src/strategy/placement.ts` 中增加：

```ts
export type AvoidFourthGoal =
  | "none"
  | "winOut"      // 抢和结束
  | "tenpaiKeep"  // 保听到流局
  | "fold"        // 防放铳
  | "chase";      // 自己四位，需要脱四
```

并扩展 `PlacementAdjustment`：

```ts
export interface PlacementAdjustment {
  context: PlacementContext;
  pushBias: number;
  defenseWeightMul: number;
  valueWeightMul: number;
  shantenWeightMul: number;
  ukeireWeightMul: number;
  avoidFourthGoal: AvoidFourthGoal;
  reasons: Reason[];
}
```

当前实现里只有 `defenseWeightMul` 和 `valueWeightMul`。本方案需要新增：

- `shantenWeightMul`：保听/速度策略下避免向听后退。
- `ukeireWeightMul`：保听/抢和策略下鼓励最快进张。

`applyPlacementPolicy` 需要同步乘上这两个字段：

```ts
function applyPlacementPolicy(policy: NanikiruPolicy, placement: PlacementAdjustment): NanikiruPolicy {
  return {
    ...policy,
    shantenWeight: policy.shantenWeight * placement.shantenWeightMul,
    ukeireWeight: policy.ukeireWeight * placement.ukeireWeightMul,
    defenseWeight: policy.defenseWeight * placement.defenseWeightMul,
    valueWeight: policy.valueWeight * placement.valueWeightMul,
  };
}
```

## 需要传入手牌进度

当前 `evaluatePlacementAdjustment(state)` 只看局况，不看向听数。南四避四必须知道 `shanten`，否则无法区分：

- 听牌：抢和结束。
- 一向听：保听/速度。
- 三向听起手：早巡不应弃和。
- 终盘手慢：防放铳。

建议改成：

```ts
evaluatePlacementAdjustment(state, {
  shanten,
  highValueHand,
});
```

类型：

```ts
export interface PlacementInput {
  shanten: number;
  highValueHand: boolean;
}
```

`chooseAction` 中已有 `preliminary.shanten` 和 `highValueHand`，可以直接传入。

## 南四微差避四判定

### 终局定义

第一版建议：

```text
isFinalRound = bakaze === "2z" && kyoku === 4
isFinalOrNearFinal = bakaze === "2z" && kyoku >= 3
```

当前代码中 `isFinalOrNearFinal` 为南场且 `kyoku >= 3`，可以保留，同时新增更严格的 `isFinalRound`。

### 微差定义

三位领先四位：

```text
leadOverFourth = selfPoints - fourthPoints
```

建议分级：

```text
leadOverFourth <= 1000: critical
leadOverFourth <= 4000: close
leadOverFourth <= 8000: nearby
```

其中 `<= 4000` 是本方案重点，因为流局听牌罚符足以改变排名。

### 四位威胁

需要识别四位是否是威胁者：

```ts
fourthThreatening =
  fourthPlayer.riichi === true
  || fourthPlayer.calls.length >= 2;
```

如果四位在进攻，放铳风险的权重要额外提高，但不能无条件覆盖听牌/好一向听的抢和目标。

## 目标选择规则

### 自己是四位

自己四位时，目标不是“避四防守”，而是“脱四”：

```text
if selfRank === 4:
  avoidFourthGoal = "chase"
  pushBias += 2
  valueWeightMul *= 1.25
```

理由：

```text
当前四位，需要保留脱四所需打点推进。
```

### 南四三位微差领先

核心规则：

```text
if isFinalRound && selfRank === 3 && leadOverFourth <= 4000:
  if shanten === 0:
    goal = winOut
  else if shanten === 1:
    goal = tenpaiKeep
  else if turn <= 6:
    goal = none
  else if turn <= 11:
    goal = fourthThreatening ? tenpaiKeep : tenpaiKeep
  else:
    goal = fourthThreatening ? fold : tenpaiKeep
```

解释：

- 听牌：和牌结束是最直接避四路线。
- 一向听：保留速度，尽快听牌或和牌。
- 早巡两向听以上：不应过早弃和，只轻微谨慎。
- 中巡两向听以上：开始重视保听和速度。
- 终盘手慢且四位威胁：防放铳优先。
- 终盘四位未明显威胁：仍保留听牌罚符目标。

### 南三或南场后半三位接近四位

非南四但已接近终局时，避四压力存在，但不应像南四一样强行抢速度：

```text
if isFinalOrNearFinal && selfRank === 3 && leadOverFourth <= 8000:
  if shanten <= 1:
    goal = tenpaiKeep
  else if fourthThreatening && turn >= 8:
    goal = fold
  else:
    goal = none
```

## 各目标的权重建议

### winOut

用于南四微差且自己听牌。

```text
pushBias += 2
ukeireWeightMul *= 1.2
defenseWeightMul *= 1.1
valueWeightMul *= 0.9
```

含义：

- 速度和和牌优先。
- 仍保留一点防守，避免无意义放铳。
- 高打点不是第一目标，能结束牌局更重要。

理由：

```text
南四与四位微差，当前听牌，和牌结束是主要避四路线。
```

### tenpaiKeep

用于南四微差且需要保听到流局。

```text
shantenWeightMul *= 1.2
ukeireWeightMul *= 1.15
defenseWeightMul *= 1.15
valueWeightMul *= 0.8
```

如果 `leadOverFourth <= 1000`：

```text
shantenWeightMul *= 1.1
ukeireWeightMul *= 1.05
```

含义：

- 避免向听后退。
- 鼓励最快听牌。
- 适度保留防守，避免为了保听切超危险牌。
- 降低高打点路线权重，因为微差避四更需要速度/听牌。

理由：

```text
南四与四位微差，流局听牌也能避免罚符逆转，当前优先保听和速度。
```

### fold

用于终盘手慢，尤其四位正在进攻。

```text
defenseWeightMul *= 1.45
pushBias -= 2
valueWeightMul *= 0.85
```

若四位是威胁者：

```text
defenseWeightMul *= 1.25
pushBias -= 1
```

理由：

```text
南四手牌较慢且四位正在进攻，当前优先避免放铳落四。
```

### chase

用于自己四位。

```text
pushBias += 2
valueWeightMul *= 1.25
ukeireWeightMul *= 1.05
```

理由：

```text
当前四位，需要保留脱四所需打点推进。
```

## 早巡保护

为了避免“发完起手牌就防守”，必须加入早巡保护：

```text
if turn <= 6 && shanten >= 2:
  禁止 avoidFourthGoal 将 mode 降级为 defense
  只允许小幅 defenseWeightMul，例如 1.05
```

建议在 `adjustModeByPlacement` 中加保护条件，或在 `PlacementAdjustment` 中增加：

```ts
allowModeDowngrade: boolean;
```

更简单的第一版：

```ts
if (adjustment.context.turn <= 6 && adjustment.context.avoidFourthGoal !== "fold") {
  不执行 balance -> defense
}
```

需要把 `turn` 和 `avoidFourthGoal` 放入可访问结构中。

## 模式修正建议

当前已有：

```text
balance + pushBias >= 2 -> push
balance + pushBias <= -2 -> defense
push + pushBias <= -3 -> balance
```

建议扩展为：

```text
goal === winOut:
  balance -> push
  push 保持 push

goal === tenpaiKeep:
  不直接 defense
  balance 可保持 balance
  若 pushBias >= 2，可 balance -> push

goal === fold:
  balance -> defense
  push -> balance

goal === chase:
  balance -> push
```

注意：`tenpaiKeep` 不是 `push` 也不是 `defense`。它主要通过 `shantenWeight` 和 `ukeireWeight` 表达。

### 不应起手弃和

南四三位领先四位 3000 点，早巡三向听，无人威胁：

```text
turn = 1
shanten >= 2
goal = none
mode 不应变成 defense
```

### 听牌抢和

南四三位领先四位 3000 点，自己听牌：

```text
goal = winOut
balance -> push 或保持 push
reason 包含“和牌结束”
```

### 一向听保听

南四三位领先四位 3000 点，自己一向听：

```text
goal = tenpaiKeep
shantenWeightMul > 1
ukeireWeightMul > 1
valueWeightMul < 1
reason 包含“流局听牌”
```

### 终盘手慢且四位威胁

南四三位领先四位 3000 点，终盘两向听以上，四位立直：

```text
goal = fold
defenseWeightMul 明显提高
pushBias 降低
balance -> defense
reason 包含“避免放铳落四”
```

### 自己四位追分

南四自己四位：

```text
goal = chase
valueWeightMul > 1
pushBias > 0
reason 包含“脱四”
```

## 注意事项

- 不要把 `tenpaiKeep` 当作纯进攻。它应该鼓励最快听牌，同时保留适度防守。
- 不要把 `fold` 用在早巡起手牌上。
- 不要在自己四位时触发防守型避四。
- 与四位差距 `<= 4000` 时，流局听牌罚符可能改变排名，因此要显式产生 reason。
- 第一版不需要完整 EV，只需要让模式和权重表达正确方向。
